# Spectral Fitting pipeline (SpecFit)

This repository contains a robust Python pipeline for spectral fitting of IR spectra, specifically designed for JWST MIRI observations. It generates 0D slab models using `prodimopy`, processes them into an interpolated grid cache, and performs Bayesian parameter estimation using `UltraNest`.

## 📁 Repository Contents

* **`generate_grid.py`**: Generates a grid of 0D slab models over specified ranges of Temperature (T) and Column Density (N) using `prodimopy.run_slab`. Outputs high-resolution synthetic spectra as FITS files.
* **`build_cache.py`**: Convolves the high-resolution slab models to the resolving power of MIRI (or another instrument) and rebins them onto a standard wavelength grid. Saves the output as an optimized `.npy` cache matrix for fast interpolation during the fitting process. Supports multiprocessing.
* **`config.yaml`**: The central configuration file. Defines global model hyperparameters (velocity correction, instrument resolution), MCMC sampler settings (live points, evidence tolerance), and individual molecular profiles (cache paths, fit bounds, wavelength masks).
* **`fit_spectra.py`**: The core fitting script. Uses `UltraNest` (nested sampling) to fit the cached grid spectra to the observed MIRI data. Evaluates the posterior distributions for Temperature, Column Density, and emitting area for multiple molecules simultaneously.
* **`plot_fit.py`** (or `plot_results.py`): Generates diagnostic plots from the UltraNest posteriors, including corner plots and spectral overlay plots showing the total model and individual molecular contributions compared to the data.
* **`utils.py`**: Contains helper functions for managing configurations and parsing the grid axes from the `index.txt` files generated by `prodimopy`.
* **`h2o_template.dat`**: A template water spectrum used for dynamic velocity corrections during the fitting process.
